Opérationnel en 3 minutes

Un seul docker compose, sur Linux, Windows ou macOS.

# 1. Récupérer
git clone https://github.com/zero97436/MVP.git opsora && cd opsora

# 2. Configurer
cp .env.example .env

# 3. Démarrer
docker compose up -d --build
→ https://localhost:8443

Questions fréquentes

L'édition Community est-elle vraiment gratuite ?

Oui, sans limite de temps, jusqu'à 500 hôtes, avec toutes les fonctionnalités de supervision et l'IA locale. Aucune carte bancaire requise.

Où sont mes données ?

Chez vous. Opsora est 100 % auto-hébergé (Docker). L'IA tourne en local via Ollama — aucune donnée ne quitte votre infrastructure.

Comment fonctionne l'achat d'une licence ?

Vous payez en ligne (paiement sécurisé Stripe). Une clé de licence signée vous est délivrée immédiatement par e-mail. Vous la collez dans votre fichier .env et redémarrez — les fonctionnalités se débloquent instantanément.

Puis-je migrer depuis Centreon ou Nagios ?

Oui. Import direct des fichiers Nagios/Icinga (.cfg) avec les dépendances, ou import CSV depuis n'importe quel outil (Centreon, Zabbix, PRTG, Excel). En quelques minutes.

Que se passe-t-il si ma licence expire ?

Rien ne s'arrête : Opsora repasse simplement en édition Community. Votre supervision continue de fonctionner.
